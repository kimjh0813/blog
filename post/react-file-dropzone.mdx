웹 스토리지 서비스를 개발하기 떄문에 당연히 업로드 기능이 필요하고 Drag&Drop 방식을 통한 업로드 기능은 필수 기능이였습니다.

React에서 Dropzone에 파일을 업로드 하는경우에 정말 많이 사용되는 react-dropzone 이라는 라이브러리가 있습니다.

npm install 수만 봐도 정말 많이 사용되는 라이브러리기도하고 다양한 기능을 지원해서 저도 react-dropzone 라이브러리를 사용하려고 헀습니다.

하지만 해당 라이브러리를 사용하지않고 개발한 배경은 렌더링 때문이였습니다. 해당 라이브러는 지정해준 div 외에도 해당 div에 모든 childern에게도 이벤트를 전파했습니다.

스토리지 서비스를 개발하며 MyBox를 많이 참고했는데 mybox 렌더링이 부분적으로 필요한 컴포넌트에서만 발생하는것을 보고 최대한 렌더링부분을 많이 따라가고 싶어서 개발하게 라이브러리를 사용하지않고 개발하게 되었던거 같습니다.

한창 렌더링쪽에 많은 고민을 하던 시기라
